Open Bugs

What is broken

P1·Mobile·Open·Hit target

Active lens object tap misses and hits the pet instead on mobile

After dismissing the lens panel on mobile, tapping the expected active lens object coordinate does not reopen the panel.

A mobile user can get stuck in the core reflection loop if they close the prompt and try to resume by tapping the active lens object.

Actual: The panel stays closed and the garden reports "Your pet notices you back."

Expected: Tapping the active lens object reopens the current lens panel.

  1. Open the app at the mobile Chrome viewport.
  2. Complete onboarding.
  3. Start the lens journey from the canvas signal.
  4. Press Escape to dismiss the lens panel.
  5. Tap the expected active word lens object area.

Evidence: `corepack pnpm run e2e` failed at `e2e/garden-loop.spec.ts:79` in the `mobile-chrome` project. The isolated repro also failed. Relevant implementation areas: `src/game/GardenGame.ts:680`, `src/game/GardenGame.ts:731`, and `src/game/gardenLayout.ts:173`.

P2·Data controls·Open·Filename

Export filename can show yesterday's date outside UTC

The seed export content is correct, but the generated filename uses UTC date formatting.

Users in time zones ahead of UTC can receive an export file dated one day earlier than their local day.

Actual: In the Europe/Amsterdam June 9, 2026 session, export downloaded `signal-garden-seeds-2026-06-08.json`.

Expected: The filename should use the user's local date, for example `signal-garden-seeds-2026-06-09.json`.

Evidence: Standalone Playwright export check returned `{"suggestedFilename":"signal-garden-seeds-2026-06-08.json","includesSeed":true,"bytes":646}`. Cause is `new Date().toISOString().slice(0, 10)` in `src/App.tsx:245`.
