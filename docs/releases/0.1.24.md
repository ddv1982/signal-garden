# Release 0.1.24 — teach the garden lights to whisper

Replace random screen-space motes with anchored pond, pet, and active-lens ambience, then tune the desktop treatment so the effects read as soft garden atmosphere across light and dark themes.

## Highlights

- Anchors garden ambience to the pond and companion instead of arbitrary viewport positions.
- Adds deterministic effect layout helpers for pond shimmer, pond motes, pet ambience, and light-mode active-lens motes.
- Keeps desktop ambience richer than compact screens while reducing color intensity after visual QA.
- Adds ambience state hooks for browser and reduced-motion assertions.
- Preserves mobile calm, active-lens tap safety, and reduced-motion stillness.

## Lore

Constraint: Desktop ambience should be visible enough to make the pond, companion, and active lens feel alive without turning the painting into a field of colored UI dots.
Rejected: Keep random viewport motes | They explained the mobile-only sparkle, but they floated over arbitrary parts of the scene and made desktop feel inconsistent.
Rejected: Brighten every particle until screenshots prove the feature | That solved visibility while making the effects look pasted over the watercolor world.
Rejected: Remove desktop ambience after the first tuning pass felt prominent | That would avoid distraction, but it would lose the intended parity with the mobile garden.
Confidence: high
Scope-risk: narrow
Directive: Let the pond and companion breathe with anchored light, and let the painting keep the final word.
Tested: `pnpm typecheck`; `pnpm test -- src/game/__tests__/gardenEffects.test.ts --runInBand`; targeted Playwright ambience checks on Chromium desktop and mobile Chrome; full release verification rerun before tagging; in-app Browser screenshots at 1280x720 light and dark after the softened tuning.
Not-tested: Physical iOS Safari and macOS Safari were not available in this release pass; GitHub Actions production deploy and release publishing have not run for this tag yet.
