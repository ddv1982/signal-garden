# Release 0.1.28 — clear the lens path

Remove the extra lens journey chrome so the focused panel, compact current-lens status, and active garden prop carry the journey without a duplicate close button or unused chip rail.

## Highlights

- Removes the visible `Close Panel` control from the lens panel while keeping Escape dismissal available for keyboard users.
- Removes the unused lens action chip submenu that repeated the seven lens steps under the current status badge.
- Keeps re-entry to a dismissed in-progress lens journey through the active canvas lens object and the existing accessible start control.
- Cleans stale message keys and planning docs that still described the removed chip rail as current UI.
- Hardens Playwright coverage for the removed submenu, draft reopen behavior, mobile reopen fallback, and pet-debug reduced-motion readiness.

## Lore

Constraint: The lens journey needed less duplicate chrome without changing the seven-lens flow, persisted draft data, or the compact current-lens status that tells travelers where they are.
Rejected: Keep the `Close Panel` button as a harmless escape hatch | The screenshot made it clear the control was visual noise, and Escape already preserves a non-destructive keyboard dismissal path.
Rejected: Leave the lens action chip rail because tests used it for re-entry | Tests should follow the remaining product affordance, not keep a retired submenu alive.
Rejected: Remove `actionLabel` from lens definitions during cleanup | Seed history still uses those labels to explain completed journeys, so deleting them would turn a UI cleanup into a data-display refactor.
Confidence: high
Scope-risk: low
Directive: Let the current lens stone speak alone: one panel, one compact status, one garden prop, no redundant rail.
Tested: `pnpm typecheck`; `pnpm test` (110 tests, 17 files); targeted Playwright lens cleanup checks across Chromium and mobile Chrome; `pnpm build`; `pnpm e2e` (51 passed, 13 skipped across chromium and mobile-chrome).
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet; physical iOS Safari and macOS Safari were not available in this release pass.
