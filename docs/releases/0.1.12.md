# Release 0.1.12 — mend the sky seam

Replace the garden backdrop edge fix after visual QA found the runtime sky patch drawing as a hard horizontal splice across the top of the painting.

## Highlights

- Adds a cleaned `background-v4.webp` with chroma-contaminated edge pixels repaired in the asset itself.
- Removes the Phaser top-strip patch layer so the garden background renders as one continuous painting.
- Updates the home art, onboarding backdrop, and runtime canvas to use the same cleaned background.
- Verifies desktop and mobile captures of the rendered garden top edge, including pixel checks for pink edge contamination.

## Lore

Constraint: The garden sky must read as one hand-painted surface, with no magenta source edge and no copied strip pretending to be sky.
Rejected: Keep the runtime top patch | It hid the chroma edge, but the sampled strip drew a visible horizontal seam across the clouds.
Rejected: Add another overlay or tint | A wash could soften the seam in one viewport while leaving a broken draw model in place.
Rejected: Crop or shift the whole background | That would disturb plot, lens, signal, and pet anchors to solve an asset-edge problem.
Confidence: high
Scope-risk: narrow
Directive: Clean the painting at the asset boundary, then render it once.
Tested: `pnpm typecheck`; `pnpm test -- --runInBand`; `pnpm build`; `pnpm e2e` with 21 passed and 11 expected skips; desktop rendered-canvas pixel sampling at 1280x720 with zero pink edge pixels in the centered top band; mobile rendered-canvas pixel sampling at 390x844 with zero pink edge pixels in sampled top rows; visual review of desktop and mobile top-edge crops.
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet.
