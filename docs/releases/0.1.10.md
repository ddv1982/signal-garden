# Release 0.1.10 — quiet the garden border

Remove the thin magenta artifact that could appear along the top edge of the Phaser garden after visual QA found source-sheet color leaking from the background image edge.

## Highlights

- Crops the contaminated top strip from the garden background texture at render time.
- Preserves the existing garden cover sizing so plot placement, lens objects, pet positioning, and pointer targets keep their current coordinate system.
- Keeps the fix scoped to the background draw path instead of tinting the whole scene or changing the garden shell border.
- Verifies the corrected edge on desktop and mobile rendered garden captures.

## What changed

- Release metadata:
  - `package.json`
  - `docs/releases/README.md`
  - `docs/releases/0.1.10.md`
  - Bumps the package release line to `0.1.10` and adds this note to the release index.
- Garden rendering:
  - `src/game/GardenGame.ts`
  - Adds a named top-trim constant for `background-v3.webp`, crops the dirty source pixels before drawing, and restores the intended display size after the crop.

## Lore

Constraint: The garden background should feel hand-painted at the edges, not reveal source-sheet magenta where the canvas meets the app frame.
Rejected: Change the app shell border color | The sampled artifact was baked into the garden background pixels, not the CSS border.
Rejected: Hide the line with a scene-wide tint | A broad overlay would dull the whole garden while leaving the asset contamination unowned.
Rejected: Reposition all garden coordinates around a shorter background | Plot, lens, signal, and pet placement already use the full design frame; the fix trims only the rendered source pixels and keeps display sizing stable.
Confidence: high
Scope-risk: narrow
Directive: Let the garden keep its painted sky, but never let chroma-edge residue become part of the visible frame.
Tested: `pnpm typecheck`; `pnpm test -- --runInBand`; `pnpm build`; Playwright desktop rendered-canvas pixel sampling at 1280x720; Playwright mobile rendered garden capture at 390x844; visual review of desktop and mobile garden captures.
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet.
