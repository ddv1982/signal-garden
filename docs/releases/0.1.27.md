# Release 0.1.27 — tend the garden paths

Clean up the garden's supporting paths without changing the visible journey: stronger accessibility behavior, clearer React state ownership, safer Phaser redraw cleanup, and leaner CI validation around the generated message catalog.

## Highlights

- Adds keyboard navigation for seed-detail tabs and modal focus/inert handling for onboarding.
- Moves garden data and settings persistence out of `App` into focused hooks with component-level tests.
- Clears stale per-draw Phaser dataset markers, including signal motion metadata, before redraws.
- Replaces positional `GardenGame.update(...)` calls with a named update object and unregisters scene lifecycle work on shutdown.
- Compiles Paraglide messages once in CI before using compiled validation scripts, while keeping standalone public scripts self-contained.
- Refreshes garden asset notes so runtime documentation matches the checked-in browser assets.

## Lore

Constraint: The garden needed quieter caretaking, not a new shape; storage keys, player-visible flows, Phaser visuals, and release gates had to keep their paths.
Rejected: Rewrite `App` into new view layers in one sweep | The largest component still needs further tending, but moving persistence side effects first gave safer seams and tests without a risky JSX migration.
Rejected: Replace the full Phaser redraw model during cleanup | The safer step was to make redraw cleanup explicit and stop retained listeners/timers; retained layers can come later with visual comparison time.
Rejected: Make generated Paraglide output checked in to avoid repeated compiles | That would trade CI time for generated-file churn, so CI now compiles once while public scripts remain self-contained.
