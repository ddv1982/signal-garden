# Release 0.1.25 — give each garden spirit its own tools

Split the large Phaser garden scene into focused renderers for seeds, planting, lens objects, ambience, and landmarks, while tightening the seams that keep pending seeds, mobile planting, and release verification honest.

## Highlights

- Moves seed rendering, seed hit-testing, and seed watering/planting animation out of `GardenScene`.
- Moves pending-seed drawing, plot markers, drag/drop planting, and plant-at-plot dispatch into a focused planting renderer.
- Moves lens object rendering, active lens effects, hit-target dataset hooks, and reveal/farewell animation into a lens renderer.
- Moves pond ambience and garden landmarks into separate renderers while preserving draw order and reduced-motion behavior.
- Hardens pending-seed recovery and prevents a drag/drop double-plant edge case.
- Adds mobile smoke coverage to CI and removes the deprecated TypeScript `baseUrl` path configuration.

## Lore

Constraint: The garden scene needed smaller caretakers without changing what the player sees, what `GardenGame` exports, or how pending seeds survive the liminal space before planting.
Rejected: Rewrite the Phaser scene as multiple scenes | That would create a cleaner-looking boundary, but it would also risk public API churn and event-order regressions for no user-visible gain.
Rejected: Leave planting, lens, and ambience logic tangled until a later redesign | That kept today's behavior safe, but it made every future garden adjustment pay the cost of one oversized scene.
Rejected: Silence TypeScript's `baseUrl` warning with a future-version deprecation flag | The local compiler rejected the flag, and explicit relative path targets removed the warning path without waiting for TypeScript 7.
Confidence: high
Scope-risk: medium
Directive: Give each garden spirit its own tools, but keep the same gate, paths, and planted seeds underfoot.
Tested: `pnpm format:check`; `pnpm lint`; `pnpm typecheck`; `pnpm test` (104 tests, 14 files); `pnpm build`; `pnpm e2e --project=chromium` (31 passed, 1 mobile-smoke test skipped in the desktop project); `pnpm e2e:mobile-smoke`; focused Chromium planting, lens, ambience, and landmark checks during the split.
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet; physical iOS Safari and macOS Safari were not available in this release pass.
