# Release 0.1.0 — open the garden gate

Prepare Signal Garden for its first Vercel-hosted beta path by pairing a CI-owned production deploy with browser-ready app icons.

## Highlights

- Deploys production from GitHub Actions after typecheck, unit tests, Playwright smoke coverage, and production build pass.
- Keeps Vercel Git auto-deploys disabled so production releases are owned by the repository workflow.
- Adds favicon and touch-icon assets so the app has a stable browser and mobile identity on Vercel.

## What changed

- Vercel deployment:
  - `.github/workflows/ci.yml`
  - `vercel.json`
  - Production deploy now runs only on `main` pushes, using the Vercel CLI with repository secrets.
  - The Vite `dist` output and single-page app fallback stay intact.

- App identity:
  - `index.html`
  - `public/`
  - `scripts/generate-favicons.mjs`
  - Browser, PNG, ICO, Safari mask, and Apple touch icons now ship from the static public folder.

## Lore

Constraint: Preserve the local-first prototype, Vite static hosting model, and browser-owned reflection data while opening a production deploy path.
Rejected: Vercel Git auto-deploys | Space Explorer already uses CI-owned deploys, and Signal Garden should follow the same release control model.
Rejected: Runtime environment wiring | The current app does not need runtime secrets or server functions; deployment metadata belongs in GitHub secrets only.
Confidence: high
Scope-risk: narrow
Directive: Let `main` be the production garden gate; tags and notes describe the release, while CI performs the deploy after verification.
Tested: `corepack pnpm run typecheck`; `corepack pnpm run test`; `corepack pnpm run build`.
Not-tested: The live GitHub Actions production deploy has not run yet for this release tag.
