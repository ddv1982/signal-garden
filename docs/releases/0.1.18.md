# Release 0.1.18 — reveal one lens at a time and rebuild the garden's bones

Focus the lens journey down to a single visible prop with gentle hand-off transitions, refine the panel and chrome around it, and restructure the two largest source files into focused modules with a lint safety net — all without changing how the garden behaves.

## Highlights

- Reveals only the current lens prop in the garden; completing a step fades the outgoing prop down and grows the next one in after a beat, with a "Step n of 7" meter in the lens panel.
- Refines the UI chrome: stronger glassmorphism on the lens panel, pill nav and theme toggle, hover/press button states, and a ✓-prefixed progress rail for completed lenses.
- Adds ambient life — drifting motes (pollen by day, fireflies at dusk) and a soft focus vignette while a lens is active, all gated behind reduced motion.
- Decomposes `App.tsx` (1,114 → 455 lines) into `useLensJourney`, `useWateringForm`, `useSystemTheme`, and `SeedDialog`/`LensPanel`/`OnboardingPanel`/`TopBar` components.
- Splits `GardenGame.ts` (2,063 → 79 lines) into a `GardenScene`, a `PetController` owning the companion's animation state machine, and data modules for sequences, textures, and dark-lens lighting.
- Introduces ESLint + Prettier enforced in CI; the first lint pass surfaced and fixed four latent React issues, including a callback ref updated during render and setState-in-effect patterns.
- Makes persistence failures observable: corrupted data and storage-quota errors now warn instead of failing silently, and a migration seam preserves user data across future schema bumps.

## Lore

Constraint: Ship the calmer one-lens-at-a-time journey while cutting the two largest files down to focused modules, without changing a single observable behavior — every phase gated on typecheck, 86 unit tests, and the full Playwright suite.
Rejected: Adopt the official Phaser template's EventBus bridge now | The existing callback bridge is sound once the ref update moved out of render; two Phaser→React signals don't justify a new communication layer.
Rejected: Introduce a state-management library | Local state with reducer-backed custom hooks covers every flow in the app; a store would add indirection without removing any complexity.
Rejected: Extract a full GardenRenderer from the scene | After the pet controller, texture manifest, and lighting tables moved out, the scene is a single rendering concern; further splitting traded clarity for plumbing.
Rejected: Name every inline hex color into a palette | Single-use scene-art colors read better next to the shapes they paint; only the structured lighting tables and theme texture lookup earned extraction.
Confidence: high
Scope-risk: medium
Directive: Reveal one lens at a time, and leave the garden's code as tidy as its beds.
Tested: `pnpm lint`; `pnpm typecheck`; `pnpm test` (86 tests, 11 files); `pnpm e2e` (47 passed, 11 skipped — matching the pre-refactor baseline on desktop and mobile Chromium); `pnpm build`; `pnpm lens:audit`; `pnpm pet:audit`; `pnpm garden:audit:dark`; production bundle compared against the 0.1.17 baseline (main chunk +1 kB gzipped, Phaser chunk byte-identical).
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet; a manual full seven-lens journey on a physical mobile device in both themes remains on the beta checklist.
