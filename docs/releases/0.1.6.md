# Release 0.1.6 - quiet the source-map path

Stop production browsers from requesting unavailable source-map files after Vercel returns `403` for deployed `.map` assets.

## Highlights

- Disables production source-map emission by default so deployed JavaScript no longer advertises `.map` files to browser developer tools.
- Keeps an explicit `BUILD_SOURCE_MAPS=true` escape hatch for hidden source maps when a private error-monitoring upload flow needs them.
- Documents the source-map policy in the project README.
- Verifies that the default production build emits no `.map` artifacts and no `sourceMappingURL` comments.

## What changed

- Release metadata:
  - `package.json`
  - `docs/releases/README.md`
  - `docs/releases/0.1.6.md`
  - Bumps the package release line to `0.1.6` and adds this note to the release index.
- Production build output:
  - `vite.config.ts`
  - Source maps now default to `false`; setting `BUILD_SOURCE_MAPS=true` switches Vite to hidden source maps without browser-visible source-map comments.
- Documentation:
  - `README.md`
  - Records when to opt into hidden maps and why the public deployment does not emit browser-referenced maps.

## Lore

Constraint: Production developer tools should not report missing source maps as application failures.
Rejected: Serve public source maps from Vercel | That would silence the console, but it would also expose source code artifacts that are not needed for this beta.
Rejected: Leave `sourcemap: true` and ignore the warning | The browser will keep requesting files the deployment is not meant to serve.
Confidence: high
Scope-risk: narrow
Directive: Let production bundles stay quiet by default, and let private diagnostics ask for maps deliberately.
Tested: `pnpm run typecheck`; `pnpm run test`; `pnpm run build`; `find dist -name '*.map' -print`; `rg -n "sourceMappingURL" dist || true`; `BUILD_SOURCE_MAPS=true pnpm exec vite build --emptyOutDir`; `pnpm exec playwright install chromium`; `pnpm run e2e --project=chromium`.
Not-tested: GitHub Actions production deploy has not run for this tag yet.
