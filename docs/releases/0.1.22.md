# Release 0.1.22 — gather the garden's words into one seed packet

Move Signal Garden's user-facing copy out of scattered component and domain literals and into one typed Paraglide message catalog. The app still ships English only, but every runtime label now has a single home and a compile-time path toward translation.

## Highlights

- Adds Paraglide JS and an Inlang project with `messages/en.json` as the source of truth for app copy.
- Replaces runtime UI, accessibility, domain, date, validation, pet-message, and Phaser canvas text literals with typed `m.*()` message functions.
- Keeps generated `src/paraglide/` output out of git, while making `build`, `typecheck`, `test`, and `e2e` compile it first so clean checkouts and CI do not depend on local generated state.
- Pins the Inlang message-format plugins as dev dependencies and loads them from local package files instead of a CDN during compile.
- Updates unit and Playwright assertions to read expected labels from the message catalog, reducing brittle duplicated copy in tests.
- Documents the copy-editing path in the README and records the completed migration plan in `docs/i18n-labels-plan.md`.
- Adds small generated-output hygiene for lint and formatting by ignoring `src/paraglide/`.

## Lore

Constraint: Runtime copy had to become editable in one place without turning the React app, domain helpers, and Phaser scene into separate translation systems.
Rejected: A plain TypeScript copy dictionary | It would centralize English, but it would leave plural rules, parameter typing, and future locale generation as custom infrastructure.
Rejected: `react-intl` or `react-i18next` | The copy lives outside React too: domain helpers, persistence-adjacent formatting, tests, and the Phaser scene all need plain importable functions.
Rejected: Commit generated Paraglide output | It would make clean checkouts simple, but it would also add generated churn to every copy edit; precompiling in scripts gives CI the same reliability without tracking output.
Rejected: Load Inlang plugins from jsDelivr at build time | The default works locally, but release builds should come from the lockfile, not a CDN fetch.
Confidence: high
Scope-risk: medium
Directive: Let the garden keep speaking English today, but make every word know where it lives.
Tested: `rm -rf src/paraglide && pnpm typecheck`; `pnpm lint`; `pnpm format:check`; `pnpm test` (96 tests, 13 files); `pnpm build`; `pnpm e2e` (47 passed, 11 skipped across chromium and mobile-chrome); browser preview at 390, 768, and 1280 px with no horizontal overflow and rendered canvas.
Not-tested: Additional locales were not added; no runtime locale switcher was built; Paraglide plural variants were not introduced yet because English-only `_one`/`_many` message pairs cover the current release; GitHub Actions production deploy and release publishing have not run for this tag yet.
