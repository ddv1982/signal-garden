# Release 0.1.3 — keep the gate for real journeys

Remove the local-only pet animation review loop from the CI production deploy gate after repeated GitHub runners showed it could block an otherwise healthy deploy.

## Highlights

- Keeps CI e2e coverage for the garden journey, persistence, planting, watering, reduced motion, and pet behavior smoke.
- Skips only the dev-only pet-debug pose review harness in CI.
- Leaves the pet-debug review test runnable locally for visual QA.

## What changed

- E2E deploy gate:
  - `e2e/garden-loop.spec.ts`
  - The local visual animation review loop now skips when `CI` is set.
  - User-facing pet behavior remains covered by the no-bounce, sleep, and reduced-motion sequence tests.

## Lore

Constraint: Production deploy should stay gated by user-facing browser behavior, not by a local animation-review harness.
Rejected: Remove e2e from deploy | The real garden flows still need to pass before Vercel receives production.
Rejected: Keep rerunning the same failed gate | The logs showed the same non-user-facing debug test blocking deploy repeatedly.
Confidence: high
Scope-risk: narrow
Directive: CI protects the garden path; local visual QA can still inspect every pet pose.
