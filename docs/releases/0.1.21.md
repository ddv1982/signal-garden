# Release 0.1.21 — settle the planting beds into the painted ground

Redesign every desktop planting spot so seeds land in deliberate garden beds instead of scattering across the scene — and teach the plots that the painted garden slides under the window. Several spots used to sit on the pond water, the walking path, the stone mosaic, the companion cat, or a lens prop, and which ones depended on how wide your window happened to be.

## Highlights

- Rearranges all 12 desktop planting spots into recognizable beds: a terraced two-row bed on the left foreground, a quiet pair on the upper-left slope, a sprout row along the pond's bottom rim, and a run of plots climbing the right bank toward the gate.
- Fixes the placement collisions in the old layout — `middle-center-right` floated on the pond, `back-left` sat on the word-stones lens prop, `middle-left` and `front-left` overlapped the cat, and `front-center` blocked the walking path.
- Accounts for the cover-cropped background: plot coordinates are viewport-relative while the artwork is center-cropped, so a spot that was clean at one window width landed on the mosaic or the gate path at another. Each plot now carries a narrow and a wide anchor, hand-tuned against the artwork at visible crop ratios 0.45 and 0.68, and `createGardenPlots()` interpolates between them from the frame's actual crop ratio.
- Keeps every plot id unchanged, so seeds already planted in saved gardens keep their homes; the mobile plot set is untouched.
- Threads the canvas height through `availableGardenPlots()` / `firstAvailableGardenPlot()` so the scene resolves frame-accurate positions instead of assuming an uncropped frame.
- Hardens the layout tests: pool-water and walking-path avoidance now hold across six canvas sizes instead of one, and a new test keeps plots clear of the resting pet and the signal orb.

## Lore

Constraint: Plot ids are persisted inside saved seeds and pinned by tests — every spot may move, none may be renamed; the mobile plot set stays untouched.
Rejected: Anchor plots to the artwork's design coordinates | The 3:1 panorama is center-cropped at common window shapes, so art-anchored beds fall off-screen exactly when the window narrows; viewport anchoring is what keeps every plot reachable.
Rejected: A single retuned static coordinate set | Verified on-screen that coordinates tidy at a 0.68 crop ratio land on the mosaic and the gate path at 0.46 — no one set is clean across the realistic crop range.
Rejected: Discrete aspect-ratio buckets | Plots and planted seeds would jump between sets mid-resize; interpolation moves them smoothly through positions verified at both ends and spot-checked between.
Confidence: high
Scope-risk: low
Directive: Plant where the painting offers ground, and let the beds breathe with the window.
Tested: `pnpm lint`; `pnpm typecheck`; `pnpm format:check`; `pnpm test` (96 tests, 13 files); `pnpm build`; `pnpm e2e` (47 passed, 11 skipped across chromium and mobile-chrome); browser-verified with a seeded twelve-plot garden at 1440, 1250, and 1050 widths in dark theme, with marker-overlay passes against the pool, paths, mosaic, pet, signal, lantern, vine, dream-stone, and all seven lens props.
Not-tested: Light-theme visuals were not re-shot (plot geometry is theme-independent and the light background shares the composition); asset audits were not rerun (no asset changes); GitHub Actions production deploy and release publishing have not run for this tag yet; physical-device passes remain on the beta checklist. Known residual quirks left as-is: the vine landmark stands in the pond at wide crops, and the design-anchored dream-stone can drift over right-edge plots at near-square windows with seven or more seeds.
