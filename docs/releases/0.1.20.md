# Release 0.1.20 — sweep the prototype shapes from the painted garden

Remove the faint horizontal line that cut across the garden in both themes, then retire the entire pre-asset procedural drawing era it belonged to: every unreachable fallback branch, an orphaned texture nothing ever drew, and a new test that keeps the painted asset set and the scene honest with each other.

## Highlights

- Fixes the subtle line visible across the garden in light and dark mode — a prototype-era "horizon glow band" still rendering on top of the painted background.
- Deletes every dead procedural fallback: the five-shape background composition, the lantern/vine/dream-stone landmark shapes, the stem-and-petals seed drawing, the lens focus circles, and the fifteen-shape primitive cat. All textures are statically bundled and preloaded, so none of these branches could run.
- Tightens the contracts the fallbacks had loosened: `addPropImage` always returns an image, `PetController.attach` requires a real sprite, and `seedPropKey` returns `string` instead of `string | null`.
- Adds a texture-coverage test asserting every texture the scene requests has light and dark variants bundled — and, in reverse, that every bundled texture is actually reachable from a scene request.
- The reverse check immediately caught `prop-soil-patch`: bundled and preloaded on every page load, drawn by nothing. Removed along with its two WebP files.
- Documents the cleanup as `docs/art-asset-cleanup-plan.md` and regenerates the garden doc screenshots without the artifact line.

## Lore

Constraint: Behavior-preserving — the painted garden must render exactly as before, minus the artifact; no new runtime guards, no visual redesign.
Rejected: Keep the procedural fallbacks as defensive rendering | Every texture is statically imported and loaded in Phaser's preload phase before create() runs, so the fallback branches were unreachable — and the one that did execute drew the very artifact this release removes.
Rejected: Guard the horizon band behind a texture check | The first fix did exactly that; deleting the band along with the rest of the fallback era removes the whole class of bug instead of fencing in one instance.
Rejected: Leave prop-soil-patch bundled for future use | Nothing has ever drawn it, it cost a network load on every visit, and the new reachability test exists precisely to fail on silent passengers like this.
Confidence: high
Scope-risk: low
Directive: Trust the painted garden; carry no ghost shapes for a world that no longer needs them.
Tested: `pnpm lint`; `pnpm typecheck`; `pnpm format:check`; `pnpm test` (95 tests, 13 files); `pnpm build`; `pnpm lens:audit`; `pnpm pet:audit`; `pnpm garden:audit:dark`; `pnpm e2e` (47 passed, 11 skipped across chromium and mobile-chrome); browser-verified light and dark themes with a seeded seven-plant garden covering every reworked draw path (background, lantern, vine, dream-stone, seed stages, lens props, companion).
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet; physical-device passes remain on the beta checklist.
