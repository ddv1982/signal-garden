# Release 0.1.26 — give the garden gate a faster engine

Move the project onto the latest Vite 8 stack, replace deprecated chunking config with Rolldown-native code splitting, and refresh the README badges so the public garden signposts match the tools now carrying the app.

## Highlights

- Upgrades the app stack to current React, Phaser, Vite, TypeScript, Vitest, Playwright, ESLint, and related tooling releases.
- Replaces deprecated Vite 8 `rollupOptions`/`manualChunks` usage with `rolldownOptions.output.codeSplitting.groups` for the isolated Phaser runtime chunk.
- Keeps Phaser lazy-loaded as its own garden-runtime chunk while accepting the existing non-fatal chunk-size warning.
- Adds the narrow font module declaration needed by TypeScript 6 side-effect import checks.
- Makes date-format tests locale-aware so release verification follows the runtime locale instead of assuming English month names.
- Refreshes README badges for TypeScript, React, Phaser, and Vite.

## Lore

Constraint: The garden needed the newest toolchain without letting a bundler migration redraw the player-facing world or hide deprecated config behind a passing build.
Rejected: Keep `build.rollupOptions` because Vite still accepts it | That would pass today while leaving the release on a deprecated bridge Vite 8 already points away from.
Rejected: Split Phaser into smaller manual chunks during the dependency pass | The current lazy game-runtime boundary is intentional, and deeper Phaser surgery would mix performance design with a toolchain release.
Rejected: Pin tests to English month names | The app uses the runtime locale, so the tests should verify that contract instead of forcing one machine's language into every release run.
