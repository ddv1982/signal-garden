# Release 0.1.13 — open the night garden

Add a system-following dark mode with manual sun and moon overrides, then give the runtime garden a coherent moonlit asset set that keeps the existing anchors and mobile layout stable.

## Highlights

- Adds `system`, `light`, and `dark` theme preference handling with backward-compatible settings migration.
- Applies the resolved theme before React hydrates so a saved or system dark preference does not first paint as light.
- Adds compact sun and moon controls in the topbar, plus a Settings control for returning to system appearance.
- Extends the garden renderer with theme-aware background, companion, growth, landmark, and lens prop textures.
- Adds dark asset generation and audit scripts so dark variants are checked directly for dimensions, alpha bounds, and transparent matte safety.

## Lore

Constraint: The garden should follow the visitor's system appearance before the first paint, while still letting them choose sun or moon manually.
Rejected: Only switch theme after React mounts | That would leave dark users with a visible light flash before the app can correct itself.
Rejected: Use CSS colors alone for night mode | The shell would darken, but the garden would still read as a daylight scene and break the fiction.
Rejected: Trust light asset audits for dark assets | The runtime loads different files in dark mode, so those siblings need their own alpha, matte, and dimension checks.
Rejected: Store only the full settings blob for first paint | A tiny dedicated preference key keeps the bootstrap script small, validates safely, and still falls back to legacy settings.
Confidence: high
Scope-risk: medium
Directive: Resolve the theme before hydration, keep the garden coordinate frame still, and make every runtime asset carry its own night variant.
Tested: `pnpm test -- --run`; `pnpm exec tsc --noEmit`; `pnpm run build`; `pnpm run pet:audit`; `pnpm run pet:audit:dark`; `pnpm run lens:audit`; `pnpm run lens:audit:dark`; `pnpm run garden:audit:dark`; targeted Playwright desktop and mobile theme tests for system, manual override, reload, and first-paint bootstrap behavior.
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet.
