# Release 0.1.15 — deepen the night garden

Push dark mode from blue dusk into a clearer nighttime garden, with a visible moon in the backdrop and darker foreground assets that no longer read as daylight cutouts.

## Highlights

- Darkens the garden backdrop substantially so dark mode reads as nighttime instead of an in-between dusk wash.
- Adds a painted moon, moon halo, sparse stars, and a stronger vignette while preserving the existing `1480x484` background frame.
- Centers and sizes the moon so it remains visible in both desktop and narrow mobile garden crops.
- Retunes companion and lens dark-asset profiles to reduce warm daylight values and better match the deeper night backdrop.
- Adjusts the dark scene wash and signal colors so runtime Phaser elements sit more naturally over the moonlit scene.

## Lore

Constraint: Dark mode should read as true nighttime with a moon, while the garden's Phaser frame, anchors, texture names, and mobile crop behavior stay stable.
Rejected: Keep the dusk-like background | It technically darkened the app, but the scene still felt like dim afternoon instead of night.
Rejected: Put the moon near the right-side sky opening | It looked good on desktop, but mobile center-cropping clipped the moon at the edge.
Rejected: Brighten the foreground to preserve daylight detail | That made the cat and lens props look pasted over the night garden.
Rejected: Redraw every asset from scratch | A full redraw could improve style, but it risks pose drift, alpha drift, and anchor drift for a focused visual release.
Confidence: high
Scope-risk: medium
Directive: Make the night unmistakable, keep the coordinate frame still, and tune foreground assets to belong under moonlight.
Tested: `pnpm run pet:audit:dark`; `pnpm run lens:audit:dark`; `pnpm run garden:audit:dark`; `pnpm run build`; asset dimension checks for light/dark background, companion, and lens pairs; desktop and mobile in-app browser visual checks for dark garden rendering, moon visibility, dark texture mode, and mobile overflow.
Not-tested: GitHub Actions production deploy and release publishing have not run for this tag yet.
