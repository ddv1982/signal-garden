# Release 0.1.4 - clear the mobile lens path

Fix the QA findings that blocked mobile sign-off: active lens objects now expose and use a stable mobile hit target, and seed exports name files with the user's local calendar date.

## Highlights

- Keeps the mobile active lens target clear of the pet interaction zone and bottom panel.
- Updates mobile e2e coverage to tap the computed active lens target instead of stale canvas fractions.
- Changes seed export filenames from UTC dates to local dates.
- Adds the QA report artifact for the desktop/mobile pass that found the issues.

## What changed

- Mobile garden input:
  - `src/game/gardenLayout.ts`
  - `src/game/GardenGame.ts`
  - Active lens hit targets and pet interaction geometry now share the same layout source.
  - Mobile lens placement reserves a wider right-side lane to avoid pet hit-target collisions.
- Export data:
  - `src/domain/exportSeeds.ts`
  - `src/App.tsx`
  - Export filenames now use `getFullYear`, `getMonth`, and `getDate` instead of UTC ISO slicing.
- QA and tests:
  - `e2e/garden-loop.spec.ts`
  - `src/game/__tests__/gardenLayout.test.ts`
  - `src/domain/__tests__/exportSeeds.test.ts`
  - `docs/qa/report/index.html`

## Lore

Constraint: Mobile taps should follow the visible active lens object, not a stale coordinate from an older layout.
Rejected: Move every mobile lens back to the old test coordinate | The current UI already presents one active lens at a time; the right fix is to expose and protect the real target.
Rejected: Keep UTC export filenames | Local browser data should use the user's local calendar day for downloaded files.
Confidence: high
Scope-risk: narrow
Directive: Let the garden's visible signal own the tap target, and let local exports speak in local dates.
