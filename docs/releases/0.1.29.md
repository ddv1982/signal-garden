# Release 0.1.29 — measure the garden paths

Add a clearer coverage lantern to the beta path: numeric Vitest coverage reporting, Settings data-control e2e checks, and focused lens journey tests for the hook and panel seams.

## Highlights

- Adds `pnpm coverage` with Vitest's v8 provider and source-focused text-summary output.
- Excludes generated Paraglide output and static assets from coverage totals so the report measures app code instead of catalog/build noise.
- Covers Settings seed export, delete cancel/confirm, and active lens profile reset flows in Playwright.
- Adds focused tests for `useLensJourney` profile creation, wrong-lens guidance, journey clearing, and final seed handoff.
- Adds focused `LensPanel` tests for textarea focus, Escape dismissal, submit value handling, final action text, and the rest action.

## Lore

Constraint: The garden needed better proof without moving the stones; this pass could add measurement and checks, but it could not change storage keys, lens flow behavior, or visible Settings actions.
Rejected: Treat raw file-count coverage as enough | The missing signal was workflow confidence, so Settings data controls and lens journey seams earned targeted checks.
Rejected: Count generated messages and image assets in the new coverage baseline | That would make the number noisier without telling us whether app behavior is protected.
Rejected: Add a broad UI testing dependency just for these checks | The existing React DOM/Vitest style already covers the hook and component seams with less tooling weight.
Confidence: high
Scope-risk: low
Directive: Place measuring twine along the paths, then keep walking the same garden.
